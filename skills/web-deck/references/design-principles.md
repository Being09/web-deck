# PPT 设计原则：逻辑化与视觉化

本 skill 的灵魂。所有内容决策和视觉决策都要回到这两大原则——**形式永远服务于功能**。

Phase 1.2 搭逻辑骨架时遵循"逻辑化"；Phase 3 映射 slide 布局、补缺失布局时遵循"视觉化"。

---

## 🧠 逻辑化设计原则：构建清晰的内容骨架

逻辑化是 PPT 的"内功"，决定内容能否被清晰、有说服力地传达。

### 1. 明确沟通目标与核心问题

动手前先想清楚：**"我要讲什么？"** 和 **"为什么它重要？"**。PPT 的本质是沟通的艺术，要围绕解决实际问题来构建。

一句话检验：如果不能用一句话说清"听众听完应该___"，说明目标还没想清。

### 2. 故事结构：经典三步法

**问题 → 原因 → 方案**。这是最普适的说服结构：

- **问题**：现状有什么痛点？用数据/场景让观众产生共鸣。
- **原因**：为什么会出现这个问题？挖到根因，不只停留在表象。
- **方案**：怎么解决？给出可行动的答案 + 预期效果。

适合：提案、分享、教育、产品发布。

### 3. 金字塔原理：结论先行

自上而下、结论先行，让逻辑一目了然：

```
        [核心结论]              ← 顶层：一句话主张
       /    |    \
  [论点1] [论点2] [论点3]      ← 中层：3 个左右分论点支撑
   / \     / \     / \
 [事实] [事实] [事实] [事实]   ← 底层：数据/案例/引用佐证
```

适合：汇报、论证、说服、商业计划。每张 slide 对应金字塔的一个节点。

### 4. 六项原则（让创意更有黏性）

检查你的内容是否满足（不必全中，但要有意识地选择）：

| 原则 | 含义 | PPT 应用 |
|------|------|----------|
| **简单** | 找到核心，删掉一切非必要 | 一张 slide 一个观点 |
| **意外** | 打破预期，吸引注意 | 开篇用一个反常识数据/问题 |
| **具体** | 可感知、可想象 | 用具体案例代替抽象描述 |
| **可信** | 可验证、有出处 | 数据标来源、引用权威 |
| **情感** | 让人在乎 | 讲人、讲故事，不只讲数字 |
| **故事** | 容易记住 | 用叙事线索串起 slide |

### 5. 严谨的设计流程

按 **"明确需求 → 拟定大纲 → 准备素材 → 美化设计"** 推进。避免边做边想——本 skill 的 Phase 1 就是前两步，Phase 3 才是美化。

## 🎨 视觉化设计原则：让逻辑一目了然

视觉化是 PPT 的"外功"，把逻辑清晰地"秀"出来，降低观众认知负荷。

### A. 信息设计

| 原则 | 怎么做 | 反例 |
|------|--------|------|
| **信噪比最大化** | 去除一切无关装饰，只保留传递核心信息的内容 | 给每个词加阴影、到处用 emoji |
| **图效优势** | 用图片/图标/图表代替纯文字，更易记住 | 把数据写成一段话 |
| **数据可视化** | 用图表直观展示趋势/关系/对比 | 表格里塞 50 行数字 |

**数据→图表选择速查**：
- 比较 → 柱状图
- 趋势 → 折线图
- 占比 → 饼图/环形图
- 关系 → 散点图
- 流程 → 流程图/箭头
- 层级 → 金字塔/树图

### B. 排版布局：CRAP 四原则 + 留白

| 原则 | 怎么做 |
|------|--------|
| **留白** | 大胆用空白，让主要内容"呼吸"，更聚焦。低密度模式尤其重要 |
| **亲密性** | 相关内容组织在一起（间距小），不相关的拉开（间距大），暗示分组 |
| **对齐** | 元素严格对齐带来秩序感——左对齐/居中/网格对齐，全文统一 |
| **对比** | 用大小/颜色/字重制造对比来突出重点（大数字 vs 小说明）|
| **重复** | 统一标题样式、字体、配色、装饰元素，保持全 deck 风格一致 |

### C. 视觉层级（Visual Hierarchy）

> 视觉层级是"对比"原则的深化：不只"有对比"，而是用对比**规划出一条观众目光的行走路径**。PPT 在几秒内被扫读，观众来不及读每个字——层级决定了他们**先看到什么、再看到什么、可以跳过什么**。

**建立层级的 5 个维度**（协同使用，单维不够）：

| 维度 | 作用 | PPT 应用 |
|------|------|---------|
| **大小** | 最直接的优先级信号 | 主标 36pt+、副标 22pt、正文 16pt——相邻层差不小于 1.5× |
| **颜色** | 独立于大小的注意力抓取 | 一个饱和强调色 vs 中性背景，引导视线到关键信息 |
| **对比度** | 元素与背景的可读性差 | 高对比保证关键信息秒读；低对比文本会拖慢整张节奏 |
| **间距** | 用空白"框"出重点 | 关键数字周围留白比加边框更聚焦 |
| **位置** | 顶左角是目光第一落点 | 最重要的信息放顶左，不论用哪种阅读模式 |

**单一视觉重心原则（信噪比的执行准则）**：

每张 slide 应有**且只有一个**视觉重心——观众扫一眼就抓到的那个点。

- 每张 slide 的强调点（加粗/高亮/放大）**≤2 处**，且不连续出现
- **强调泛滥 = 没强调**：处处加粗等于处处不加粗，多重心反而稀释单点冲击
- 自检：眯起眼看 slide，第一个抓住你的元素是不是你想让观众最先看到的那个？

**内容密度均衡原则（对齐与重复的执行准则）**：

同级元素应**视觉等重**——4 个格子里的文字量应接近，4 行论证的长度应对齐。

- 同级文本块字数差异 ≤30%（**计算方式：`(max - min) / 平均值 ≤ 0.3`**，相对平均值算），否则重的一侧会"压"住轻的一侧
- 同级元素字号、行高、内边距必须一致（重复原则的量化版）
- 修正方向：先按目标字数（取当前平均值或设计意图值）**砍长不补短**——长的精简到目标，短的不要硬凑字数
- 自检：眯眼扫一遍，有没有哪一格/哪一行明显比其他"重"？

**强调点的计数规则**（避免"≤2 处"被误用）：
- "强调点"指**为突出而额外加的视觉权重**：加粗关键词、颜色高亮、放大某个词
- **结构性样式不算强调点**：标题本就该大就该粗（L1/L2 字号 + 700 字重是层级，不是额外强调）、整段基础字重 400 也不是
- **颜色强调也算**：用饱和强调色标关键词 = 1 处强调，与加粗、放大一起计入 ≤2
- 所以"≤2 处"针对的是**正文区为突出某些关键词而额外加的标记**，不是统计全 slide 有几个粗体元素

### D. 字阶系统（Typographic Scale）

> 字号不能拍脑袋。建立一套全 deck 复用的字阶层级，让"这是标题/这是正文/这是注释"在视觉上一目了然。

#### 关键前提：区分"演示场景"和"阅读场景"

**字号下限取决于观众怎么消费内容——同样的字阶角色，在两种场景下字号差很多。**

| 维度 | 演示场景（PPT/keynote/视频录屏） | 阅读场景（Web/报告/手册） |
|------|--------------------------------|------------------------|
| 观看距离 | 远（投屏 1-3 米、视频播放） | 近（凑屏幕 30-50cm） |
| 消费方式 | 扫读秒级、听讲为主 | 精读、自定节奏 |
| **正文字号下限** | **18px+（推荐 20-24px）** | 14-16px |
| **标签字号下限** | **16px+（观众要扫到的）** | 11-13px |
| 内容密度 | 低（一图一观点，字少） | 中-高（可承载多段） |

**核心判别**：观众**必须读**的内容（正文、点题标签）必须达演示字号下限；观众**不需要读**的内容（纯装饰 chrome：页码、垂直 mono 标签、nav-hint）可保持小字（11-13px），它们本来就不指望观众读。

**反模式**：把 Web 字阶（正文 14-16px、标签 11-13px）直接搬到演示场景 = 观众看不清。演示场景的字阶要整体放大，同时**精简内容**（减少字数）而非"塞不下就缩小字"——内容塞不下时拆 slide，不缩字。

#### 4 层字阶 + 1 强调层

**演示场景**（PPT/视频录屏默认）：

| 层级 | 用途 | 字号（演示） | 字重 | 版面占比 |
|------|------|-------------|------|---------|
| **L1 主标** | slide 主标语、封面标题 | 48-96px（~5% 版面高） | 700 | 主导 |
| **L2 副标/章节** | topbar 标题、章节大标、引用副标 | 28-56px（~3.5% 版面高） | 700 | 支撑 |
| **L3 正文** | lede、格子正文、table 描述 | **18-24px（~1.8% 版面高）** | 400 | 主体 |
| **L4 点题标签** | kicker、点题 micro、表格表头 | **16-20px** | 600 | 观众扫读 |
| **L5 纯装饰 chrome** | 页码、垂直 mono、nav-hint、mono tick | 11-13px | 400-500 | 不需读 |
| **强调** | 关键词加粗（每张≤2 处） | 同所在层级 | 700 | 锚点 |

**阅读场景**（Web/报告/手册/异步评审）：

| 层级 | 用途 | 字号（阅读） |
|------|------|-------------|
| L1 主标 | 文章标题、报告封面 | 36-64px |
| L2 副标 | 章节标题、引用副标 | 22-36px |
| L3 正文 | 段落、列表 | 14-16px |
| L4 标签 | 标签、表头 | 11-13px |

**比例规范**：
- **相邻层级比** 1.5-2×：L1:L2 ≈ 1.7×、L2:L3 ≈ 2×、L3:L4 ≈ 1.3×
- **L3/L4 同属"可读文本层"，相邻比可放宽到 1.2-1.5×**——L3 正文（18-24px）与 L4 点题标签（16-20px）字号区间本身重叠，强制 1.5× 会逼出"正文必须 ≥24px 才能让 kicker=16 满足比例"的荒谬结论。两层的区分靠**字重 + 颜色**（L4 用 600 + 强调色）而非纯字号差
- **跨层禁止跳跃过大**：相邻层比超 2× 会断层，需要中间层过渡
- **大小字落差靠"减少大字"而非"缩小小字"**：主标过大 + 正文过小的落差，优先**降主标字号**而不是缩正文——正文已达场景下限就不能再缩
- **常见比例系数**（用 typescale 工具生成时）：1.200（minor third，温和）、1.333（perfect fourth，标准）、1.5（perfect fifth，强对比）

**多用途 deck 取严下限**：同一 deck 有多种消费方式时（如"投屏 + 异步文档"），按**最严的下限**取——既能投屏看清，也能凑近读。即取演示场景字号下限，不取阅读场景下限。

**反例诊断**：
- 如果一张 slide 看着"引语太小、主标太大" → 字阶断层，查相邻层比是否超 2×
- 如果观众反馈"看不清正文" → 正文低于演示场景下限（18px），需放大正文或精简内容
- 如果"大字和小字差别太大" → 落差过大，降主标字号让差距收敛（不要缩正文）

### E. 间距系统（Spacing Tokens）

> "留白"和"亲密性"原则的操作化。光说"大胆用空白"不够——要有一套可复用的间距 token，让全 deck 的节奏统一。

**4 档间距 token**（全 deck 复用）：

| Token | 值（参考） | 用途 |
|-------|-----------|------|
| `--gap-xs` | 8-14px | 紧邻元素：标签↔其说明、表头↔数据 |
| `--gap-sm` | 16-28px | 同组元素内：标题↔正文、kicker↔主标 |
| `--gap-md` | 28-48px | 模块间：topbar↔body、卡片间 |
| `--gap-lg` | 48-88px | 主块间：主标↔归属栏、章节间 |

**亲密性原则的间距表达**：
- 相关元素用 `--gap-xs`/`--gap-sm`（靠近 = 同组）
- 不相关元素用 `--gap-md`/`--gap-lg`（拉开 = 分组）
- **不要在两张同结构的 slide 上用不同的间距值**——节奏统一是"重复"原则的体现

**自检**：扫一遍全 deck，每张 slide 的"标题↔正文"间距是不是同一个值？如果不是，统一成 `--gap-sm`。

### F. 阅读模式：F-Pattern 与 Z-Pattern

> 根据 slide 内容密度主动选择阅读模式，而不是让目光乱跑。这是"对齐"原则在整张 slide 层面的应用。

**F-Pattern（F 型阅读）**——适合**内容密集**的 slide：

```
████████████████████  ← 顶部水平带：最常被读
██████████████        ← 中部水平带：次常读
████████              ← 左侧垂直带：常被扫
████████████████████  ← 底部：常被跳过
```

- 观众扫读文本密集 slide 时，目光走 F 形：先读顶部全宽，再读中部，最后沿左侧下行
- **底部右侧最容易被跳过**——别把关键结论放那里
- 应用：每段首词视觉强化（加粗/着色）、关键信息放左上、左侧列承载主要内容

**Z-Pattern（Z 型阅读）**——适合**极简单点**的 slide：

```
████████████████████
                   ╲
                    ╲
                     ╲
████████████████████  ← 对角线终点：行动召唤位
```

- 观众扫读极简 slide 时，目光走 Z 形：顶部水平→对角线→底部水平
- 适合：封面、章节页、引用、行动召唤
- 应用：主标放左上、视觉重心居中偏右、归属/CTA 放右下

**选择依据**：
- 文本量 > 3 段或数据表/网格 → **F-Pattern**
- 单一主标语 + 极少支撑 → **Z-Pattern**
- 中间态：按主元素位置决定（主标左上→Z；信息网格→F）

**中间态双重信号仲裁**（两个信号同时出现时）：
- **主标左上（→Z）+ 下方 ≥2 个并列卡片/网格（→F）同时成立** → **优先 F**。卡片群构成信息网格，密度信号压过单一主标信号
- **主标左上 + 下方仅 1 个支撑块** → Z（仍是单点）
- **量化定义**：信息网格 = 同结构元素 ≥2 个（2 个并列卡片即触发 F 的网格条件）；"3 段"以段落或同结构模块计（5 行指标 = 5 段，1 段含 3 个并列论点 = 1 段）
- **模糊时回流到密度判定**：眯眼看，整张 slide 视觉重心是**散布在多个卡片**（→F）还是**聚焦在一个主标语/主图**（→Z）

### G. 美学原则

- **简约至上**：少即是多，避免信息过载。一张 slide 别塞超过它能舒服承载的内容。
- **和谐统一**：色彩、字体、间距、圆角等所有视觉元素追求整体平衡。
- **东方美学（禅意）**：把**简约、自然、留白**融入设计——宁静、专业、不喧宾夺主。空的地方和满的地方同等重要。
- **装饰让位**：装饰是配角，永远为内容让出呼吸空间。装饰尺寸过大、占比过重时，主动收窄或降透明度——让它"在那里但不抢戏"。

## 💎 形式服务功能（总纲）

所有原则都指向一个核心：**形式永远服务于功能**。

- 逻辑化原则 → 保证"传达的内容是对的、有说服力的"
- 视觉化原则 → 保证"传达的方式是高效的、低认知负荷的"
- 两者结合 → 观众**听得懂、记得住、愿意行动**

### 反模式自检清单（常见错误）

**内容层反模式**：
- ❌ 为了好看加的装饰，问"它在帮传达什么？"答不出 → 删
- ❌ 一个炫酷的图表，但数据和论点没关系 → 换成合适的表达
- ❌ 字号小到看不清，为了塞下所有内容 → 拆成两张 slide
- ❌ 换了模板的字体因为"我觉得这个更好看" → 不行，字体是设计系统

**层级反模式**（视觉层级 5 维度的常见误用）：
- ❌ 所有元素同等大小、同等颜色、同等字重 → 没有层级，观众不知先看哪
- ❌ 一张 slide 里加粗了 5、6 处 → 强调泛滥等于没强调，每张 ≤2 处
- ❌ kicker 标签 13px 直接跳到主标 84px → 字阶断层，补中间层
- ❌ 把关键结论放在内容密集 slide 的右下角 → F-Pattern 下右下最易被跳过
- ❌ 4 个格子里有一格文字明显比其他多 → 同级不等重，精简或拆分

**间距反模式**（间距系统的常见误用）：
- ❌ "标题↔正文"间距在这张是 24px、在那张是 40px → 节奏不统一，用 token
- ❌ 主标和归属栏之间留了巨大空白 → 亲密性断裂，它们是一组应该靠近
- ❌ 装饰元素占了 1/3 版面，挤压内容区 → 装饰让位，收窄或降透明度
- ❌ 为了"不浪费空间"把内容塞满 → 留白比塞满更有力量

**一致性反模式**（跨 slide 的常见误用）：
- ❌ 开篇几张 slide 遵循层级原则，后面的 slide 时间紧就乱来 → 全 deck 必须一致
- ❌ 这张用左对齐、那张用居中，没有理由 → 全 deck 统一一种对齐（除非刻意对比）
- ❌ 强调色在这张是红、在那张是黄 → 强调色全 deck 用一种（模板的 accent）

## 原则 → Slide 类型映射

搭 outline 时，用这张表把"要讲什么"映射到"用什么 slide 布局"：

| 要表达的逻辑 | 推荐 slide 类型 | 主要应用的原则 | 推荐阅读模式 |
|--------------|----------------|----------------|------------|
| 开场、定调 | 封面（cover）| 简单、留白 | Z-Pattern |
| 全 deck 导航 | 目录（toc）| 亲密性、对齐 | F-Pattern |
| 抛出问题/痛点 | 大字论点（manifesto）+ 数据 | 意外、具体、对比 | Z-Pattern |
| 论证观点 | 论点 + 佐证（argument + evidence）| 金字塔、可信 | F-Pattern |
| 展示数字/成果 | 数据卡（stats）/ 大数字 | 图效优势、对比 | Z-Pattern |
| 比较 A vs B | 对比表/双栏（comparison）| 对比、亲密性 | F-Pattern |
| 流程/步骤 | 流程图/时间线（process/timeline）| 重复、对齐 | F-Pattern |
| 引用/证言 | 引用（quote）| 情感、可信 | Z-Pattern |
| 章节过渡 | 章节页（section divider）| 留白、简约 | Z-Pattern |
| 总结/行动号召 | 结尾（closing）| 简单、故事 | Z-Pattern |

这张表是建议，不是教条——**模板有什么 layout 就用什么，缺的按"六同规则"补**（见 clone-adapt-guide.md）。
