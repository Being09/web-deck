# PPT 设计原则：逻辑化与视觉化

本 skill 的灵魂。所有内容决策和视觉决策都要回到这两大原则——**形式永远服务于功能**。

Phase 1.2 搭逻辑骨架时遵循"逻辑化"；Phase 3 映射 slide 布局、补缺失布局时遵循"视觉化"。

---

## 🧠 逻辑化设计原则：构建清晰的内容骨架

逻辑化是 PPT 的"内功"，决定内容能否被清晰、有说服力地传达。

### 1. 明确沟通目标与核心问题

动手前先想清楚：**"我要讲什么？"** 和 **"为什么它重要？"**。PPT 的本质是沟通的艺术，要围绕解决实际问题来构建内容。

一句话检验：如果不能用一句话说清"听众听完应该___"，说明目标还没想清。

### 2. 故事结构：经典三步法

**问题 → 原因 → 方案**。这是最普适的说服结构：

- **问题**：现状有什么痛点？用数据/场景让观众产生共鸣。
- **原因**：为什么会出现这个问题？挖到根因，不只停留在表象。
- **方案**：怎么解决？给出可行动的答案 + 预期效果。

适合：提案、分享、教育、产品发布。

### 3. 金字塔原理：结论先行

自上而下、结论先行，让逻辑一目了然：

```
        [核心结论]              ← 顶层：一句话主张
       /    |    \
  [论点1] [论点2] [论点3]      ← 中层：3 个左右分论点支撑
   / \     / \     / \
 [事实] [事实] [事实] [事实]   ← 底层：数据/案例/引用佐证
```

适合：汇报、论证、说服、商业计划。每张 slide 对应金字塔的一个节点。

### 4. 六项原则（让创意更有黏性）

检查你的内容是否满足（不必全中，但要有意识地选择）：

| 原则 | 含义 | PPT 应用 |
|------|------|----------|
| **简单** | 找到核心，删掉一切非必要 | 一张 slide 一个观点 |
| **意外** | 打破预期，吸引注意 | 开篇用一个反常识数据/问题 |
| **具体** | 可感知、可想象 | 用具体案例代替抽象描述 |
| **可信** | 可验证、有出处 | 数据标来源、引用权威 |
| **情感** | 让人在乎 | 讲人、讲故事，不只讲数字 |
| **故事** | 容易记住 | 用叙事线索串起 slide |

### 5. 严谨的设计流程

按 **"明确需求 → 拟定大纲 → 准备素材 → 美化设计"** 推进。避免边做边想——本 skill 的 Phase 1 就是前两步，Phase 3 才是美化。

## 🎨 视觉化设计原则：让逻辑一目了然

视觉化是 PPT 的"外功"，把逻辑清晰地"秀"出来，降低观众认知负荷。

### A. 信息设计

| 原则 | 怎么做 | 反例 |
|------|--------|------|
| **信噪比最大化** | 去除一切无关装饰，只保留传递核心信息的内容 | 给每个词加阴影、到处用 emoji |
| **图效优势** | 用图片/图标/图表代替纯文字，更易记住 | 把数据写成一段话 |
| **数据可视化** | 用图表直观展示趋势/关系/对比 | 表格里塞 50 行数字 |

**数据→图表选择速查**：
- 比较 → 柱状图
- 趋势 → 折线图
- 占比 → 饼图/环形图
- 关系 → 散点图
- 流程 → 流程图/箭头
- 层级 → 金字塔/树图

### B. 排版布局

| 原则 | 怎么做 |
|------|--------|
| **留白** | 大胆用空白，让主要内容"呼吸"，更聚焦。低密度模式尤其重要 |
| **亲密性** | 相关内容组织在一起（间距小），不相关的拉开（间距大），暗示分组 |
| **对齐** | 元素严格对齐带来秩序感——左对齐/居中/网格对齐，全文统一 |
| **对比** | 用大小/颜色/字重制造对比来突出重点（大数字 vs 小说明）|
| **重复** | 统一标题样式、字体、配色、装饰元素，保持全 deck 风格一致 |

### C. 美学原则

- **简约至上**：少即是多，避免信息过载。一张 slide 别塞超过它能舒服承载的内容。
- **和谐统一**：色彩、字体、间距、圆角等所有视觉元素追求整体平衡。
- **东方美学（禅意）**：把**简约、自然、留白**融入设计——宁静、专业、不喧宾夺主。空的地方和满的地方同等重要。

## 💎 形式服务功能（总纲）

所有原则都指向一个核心：**形式永远服务于功能**。

- 逻辑化原则 → 保证"传达的内容是对的、有说服力的"
- 视觉化原则 → 保证"传达的方式是高效的、低认知负荷的"
- 两者结合 → 观众**听得懂、记得住、愿意行动**

**反模式自检**：
- ❌ 为了好看加的装饰，问"它在帮传达什么？"答不出 → 删
- ❌ 一个炫酷的图表，但数据和论点没关系 → 换成合适的表达
- ❌ 字号小到看不清，为了塞下所有内容 → 拆成两张 slide
- ❌ 换了模板的字体因为"我觉得这个更好看" → 不行，字体是设计系统

## 原则 → Slide 类型映射

搭 outline 时，用这张表把"要讲什么"映射到"用什么 slide 布局"：

| 要表达的逻辑 | 推荐 slide 类型 | 主要应用的原则 |
|--------------|----------------|----------------|
| 开场、定调 | 封面（cover）| 简单、留白 |
| 全 deck 导航 | 目录（toc）| 亲密性、对齐 |
| 抛出问题/痛点 | 大字论点（manifesto）+ 数据 | 意外、具体、对比 |
| 论证观点 | 论点 + 佐证（argument + evidence）| 金字塔、可信 |
| 展示数字/成果 | 数据卡（stats）/ 大数字 | 图效优势、对比 |
| 比较 A vs B | 对比表/双栏（comparison）| 对比、亲密性 |
| 流程/步骤 | 流程图/时间线（process/timeline）| 重复、对齐 |
| 引用/证言 | 引用（quote）| 情感、可信 |
| 章节过渡 | 章节页（section divider）| 留白、简约 |
| 总结/行动号召 | 结尾（closing）| 简单、故事 |

这张表是建议，不是教条——**模板有什么 layout 就用什么，缺的按"六同规则"补**（见 clone-adapt-guide.md）。
