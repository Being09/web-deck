# 动画设计指南：引导注意力，而非装饰

本文件是 web-deck 动画能力的灵魂。Phase 1.3 选动画强度、Phase 3 给元素加 `[data-anim]` 时读它。

核心命题：**动画是引导注意力的工具，不是装饰**。每个动画都要回答"它在帮传达什么逻辑"。纯静态让人疲劳，但处处闪烁、循环骚扰同样破坏注意力——适度、有目的的入场动画才是解。

---

## 一、动画强度四级（Phase 1.3 与用户对齐）

| 强度 | 适用场合 | 行为 | 该动的元素 |
|------|----------|------|-----------|
| **关** | 纯静态报告、无障碍优先、打印分发、严肃财务/医疗 | 无任何 `[data-anim]`，纯静态 | — |
| **轻**（默认） | 大多数演讲、汇报、分享 | 仅关键 slide 入场 | 封面/章节页主标、单张主数据、结尾 CTA |
| **适度** | 产品发布、提案、需要节奏感的分享 | 关键 slide + 结构性引导 | 上述 + 流程图描边、对比项错峰、引用浮现 |
| **丰富** | 创意/品牌/科技感场合、keynote 风演讲 | 多数 slide 有节奏的入场 | 上述 + 每张标题/数据轻入场（仍守"每元素一次、不循环骚扰"）|

**默认"轻"的依据**：多数商业/学术场合要的是清晰高效，不是炫技。"轻"恰好解决"静态疲劳"又不喧宾夺主。用户要更强或更弱，主动调整。

**强度对齐密度**：低密度/演讲主导（Phase 1.3①）与"轻/适度"天然契合——大字号 + 少量入场动画 = 戏剧张力；高密度/阅读优先通常配"关/轻"——内容密集时动画反而干扰扫读。

---

## 二、何时该动 / 何时不该动（形式服务功能的执行准则）

> 这张表是动画决策的"信噪比"——和 design-principles.md 的信噪比原则同源。该动的动，不该动的坚决不动。

### ✅ 该动（每个动画都在帮传达）

| 场景 | 为什么动 | 推荐类型 |
|------|----------|----------|
| **封面/章节页主标** | 建立 deck 基调，信号"开始了"/"进入新部分" | `fade-up` 或 `blur-in` |
| **关键数据/统计值** | 强化记忆，让数字有冲击力 | `scale-in`（大数字）+ `count-up`（滚动）|
| **流程图/时间线** | 引导视线沿路径顺序，体现"先→后"逻辑 | `draw-line`（SVG 描边）|
| **对比 A vs B** | A 先 B 后，制造张力，强化差异 | 左 `reveal-left` + 右 `reveal-right`，错峰 |
| **引用/证言** | 引文先现，出处延迟，情感节奏 | `fade-up` 引文 + 延迟 `fade-in` 出处 |
| **结尾 CTA** | 收束注意力到行动号召 | `fade-up` + 最后一个 `data-delay` |

### ❌ 不该动（动画反而破坏传达）

- **纯装饰 chrome**（页码、角标、垂直 mono 标签）——它们本来不指望观众读，动起来反而抢戏
- **正文段落整块**——动画干扰阅读，观众要读字不是看动效
- **已自解释的图表每根柱子/每个点**——图表整体一次 `fade-up` 够了，逐根动 = 噪音
- **循环闪烁/脉动**（`animation-iteration-count: infinite`）——入场动画只播一次，循环骚扰是注意力的头号杀手
- **每个元素都加 `[data-anim]`**——强调泛滥 = 没强调，呼应 design-principles.md 的"≤2 处强调"原则
- **装饰元素**（角标、纸纹、SVG 装饰）——它们是身份不是内容，静默存在即可

**自检**：眯眼看一张 slide 的动画序列——视线是不是被引导到了你想让观众先看的元素？如果到处都在动、找不到重心，砍到只剩关键的 2-3 个。

## 三、`[data-anim]` 类型 → slide 类型映射

搭完 outline（Phase 1.2）、选定强度后，用这张表把"要讲什么"映射到"用什么动画"。仿 design-principles.md 的"原则→slide 类型映射"。

| slide 类型 | 推荐动画序列 | 强度门槛 |
|------------|-------------|----------|
| **封面（cover）** | 主标 `fade-up`(delay 0) → 副标 `fade-in`(delay 2) → 归属栏 `fade-in`(delay 4) | 轻+ |
| **目录（toc）** | 标题 `fade-up`(0) → 整个目录列表 `fade-up`(2) | 轻+ |
| **章节过渡（divider）** | 主标 `blur-in`(0) → 章节号 `fade-in`(1) | 轻+ |
| **大字论点（manifesto）** | 论点 `fade-up`(0) → 支撑数据 `scale-in`(2) | 轻+ |
| **数据卡（stats）** | 大数字 `scale-in` + `count-up`(0) → 标签 `fade-up`(2) → 说明 `fade-in`(4) | 适度+ |
| **对比（comparison）** | 左项 `reveal-left`(0) → 右项 `reveal-right`(2) → 共同结论 `fade-up`(4) | 适度+ |
| **流程/时间线（process）** | 标题 `fade-up`(0) → 连接线 `draw-line`(1) → 各节点错峰 `fade-up`(2,3,4...) | 适度+ |
| **引用（quote）** | 引文 `fade-up`(0) → 出处 `fade-in`(2) | 轻+ |
| **结尾（closing）** | 主标 `fade-up`(0) → CTA `fade-up`(2) → 联系方式 `fade-in`(4) | 轻+ |

**强度门槛**：标"轻+"表示"轻"强度就建议加；"适度+"表示"适度/丰富"强度才加（"轻"强度跳过这类 slide 的动画）。

## 四、`[data-anim]` 动画类型速查

### 基础五类（5 个自带模板也用，跨模板兼容）

| 类型 | 效果 | 最适合 |
|------|------|--------|
| `fade-up` | 从下方 28px 淡入上移 | 标题、卡片、列表——最通用的"入场" |
| `fade-in` | 纯淡入，无位移 | 副标、说明、chrome 文字——不抢戏的"出现" |
| `reveal-right` | 从左向右 clip 揭示 | 左→右阅读顺序的内容、强调"展开" |
| `reveal-left` | 从右向左 clip 揭示 | 对比项的右侧、强调反向张力 |
| `scale-in` | 从 0.94 缩放淡入 | 大数字、关键图标——"聚焦"感 |

### 扩展三类（motion.css 新增，仅引 motion.css 的模板可用）

| 类型 | 效果 | 最适合 |
|------|------|--------|
| `blur-in` | 从模糊+轻微放大聚焦清晰 | 封面/章节主标——"进入"的戏剧感 |
| `draw-line` | SVG 描边从无到有 | 流程图/时间线的连接线、图表轴线 |
| `count-up` | 数字滚动增长 | 关键统计值（配 `data-count-to`/`data-count-from`）|

### CountUp 用法

```html
<div class="stat-value"
     data-anim="count-up"
     data-count-from="0"
     data-count-to="142"
     data-count-dur="1200"
     data-count-suffix="%">0%</div>
```

属性：`data-count-from`（起点，默认 0）、`data-count-to`（终点，必填）、`data-count-dur`（毫秒，默认 1200）、`data-count-decimals`（小数位，默认 0）、`data-count-prefix`/`data-count-suffix`（前/后缀如 %/M/$）。

**遵守 reduced-motion**：用户开启减少动画时，motion.js 直接显示终值不滚动。

## 五、`data-delay` 错峰节奏

入场不是齐刷刷一起出现——**错峰**制造"引导视线"的节奏。motion.css 内置 7 档（与 5 个自带模板一致）：

| `data-delay` | 延迟 | 用途 |
|--------------|------|------|
| `0` | 0s | 第一优先级：主标、大数字 |
| `1` | 0.08s | 紧随：副标、kicker |
| `2` | 0.18s | 同组次元素 |
| `3` | 0.30s | 下一组开始 |
| `4` | 0.44s | 支撑说明 |
| `5` | 0.60s | chrome、出处 |
| `6` | 0.78s | 最后收尾 |

**错峰原则**：
- **同组元素**用相邻 delay（0→1→2），表示"一起出现但有顺序"
- **不同组**跳一档以上（0→2 或 0→3），表示"先看这组再看那组"
- **视线引导**：想让观众先看的元素 delay 最小，依次递增

## 六、反模式自检清单

**内容层反模式**：
- ❌ 每个元素都加 `[data-anim]` → 强调泛滥，问"它在帮传达什么"答不出 → 删
- ❌ 循环动画（`infinite`）当常态 → 入场动画只播一次，循环骚扰是注意力杀手
- ❌ 装饰元素也加动画 → 装饰是配角，静默存在
- ❌ 正文段落整块 `fade-up` → 干扰阅读，只动标题/数据/结构元素

**时序反模式**：
- ❌ 单个动画时长 >800ms → 拖慢节奏，观众等不及
- ❌ 单个动画时长 <200ms → 显得突兀、廉价
- ❌ 全 deck 用同一 delay → 没有节奏感，用 data-delay 错峰
- ❌ 同一张 slide 7 个元素全 delay 0 → 等于没动画也没引导，分组错峰

**可访问性反模式**：
- ❌ 漏 `prefers-reduced-motion` 兜底 → motion.css 已内置全局兜底，但自加动画时记得测
- ❌ 用纯动画传达关键信息（无动画就看不到）→ 动画是增强不是依赖，静态下内容必须完整可见

## 七、强度 → 用量速查（生成时的快速决策）

| 强度 | 用哪几类 `[data-anim]` | 每张 slide 大约几个 | 重点 slide |
|------|----------------------|---------------------|-----------|
| **关** | 无 | 0 | — |
| **轻** | `fade-up` / `fade-in` | 2-3 个 | 封面、章节、结尾 |
| **适度** | + `scale-in` / `reveal-*` | 3-5 个 | + 数据卡、对比、流程 |
| **丰富** | + `blur-in` / `draw-line` / `count-up` | 4-7 个 | + 每张标题/数据轻入场 |

**总量红线**：不论强度，单张 slide 的 `[data-anim]` 元素 **≤7 个**。超过就是噪音——观众来不及看每个动画，反而疲劳。要传达更多，靠内容结构而非更多动画。

## 规范依据

- 本文件的原则是 design-principles.md"形式服务功能"在时间维度的延伸——视觉化原则（信噪比、对比、强调≤2处）同样适用于动画
- `[data-anim]`/`data-delay`/5 keyframe 约定源自 broadside/monochrome/grove/signal/studio 五个模板的既有实现，motion.css 复用同一约定以保证跨模板兼容
- `prefers-reduced-motion` 遵循 [WCAG 2.3.3 Animation from Interactions](https://www.w3.org/WAI/WCAG21/Understanding/animation-from-interactions.html)
